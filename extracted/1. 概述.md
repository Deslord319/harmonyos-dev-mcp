_# HarmonyOS MCP工具技术方案

## 1. 概述

本文档旨在为HarmonyOS应用开发设计一套完整的AI辅助开发（AI Coder）解决方案，核心是开发一个**HarmonyOS模型上下文协议（MCP）工具**。该工具将作为连接AI编程环境（如Cursor、Cline）与HarmonyOS开发工具链（DevEco Studio、hdc、hvigor）的桥梁，实现应用开发全流程的自动化和智能化。

### 1.1 设计目标

- **全流程自动化**：覆盖从代码编写、编译构建、安装部署到UI测试的全过程。
- **深度IDE集成**：通过MCP协议与主流AI IDE（Cursor、Cline）无缝集成，提供上下文感知的智能辅助。
- **解决核心痛点**：
    1. 绕开官方IDE对AI插件的限制。
    2. 实现对HarmonyOS应用的UI结构感知（DOM树拉取）。
    3. 自动化处理开发者账号和应用签名流程。
    4. 提供一个可扩展的、与HarmonyOS SDK解耦的工具层。

### 1.2 技术选型

- **AI IDE**：**Cursor** 或 **Cline**（VS Code插件），两者均提供强大的MCP客户端能力。
- **MCP Server实现**：**Python + FastMCP**，利用Python强大的胶水语言特性和丰富的库生态，集成HarmonyOS的命令行工具。
- **核心依赖工具**：
    - **hdc (HarmonyOS Device Connector)**：用于设备管理、应用安装、日志抓取和命令执行。
    - **hvigorw**：用于自动化编译、构建和打包。
    - **UITest框架**：用于UI自动化测试，是实现DOM树拉取的关键。

## 2. 系统架构

设计的核心是一个MCP服务器，它在本地运行，并与AI IDE和HarmonyOS工具链进行交互。

**交互流程**：
1.  **用户输入**：开发者在Cursor或Cline中通过自然语言提出开发需求（如“创建一个登录按钮并实现点击事件”）。
2.  **AI Agent分析**：IDE内置的AI Agent解析需求，并决定调用哪个MCP工具。
3.  **MCP工具调用**：AI Agent通过MCP协议向本地的HarmonyOS MCP Server发起工具调用请求。
4.  **工具分发与执行**：MCP Server接收请求，根据工具名称（如`get_ui_tree`、`build_app`）分发给相应的模块。工具模块通过执行`hdc`、`hvigorw`等命令行工具与HarmonyOS设备或项目文件进行交互。
5.  **结果返回**：命令行工具的执行结果被MCP Server捕获、解析，并格式化为JSON或其他结构化数据，通过MCP协议返回给AI Agent。
6.  **代码生成/操作执行**：AI Agent根据返回的结果（如UI树结构、编译错误信息）生成代码、修改文件或执行下一步操作，最终呈现给开发者。

## 3. 核心功能与Tool定义

基于`FastMCP`框架，我们将定义一系列工具（Tool）来封装对HarmonyOS的操作。以下是核心工具的定义和实现思路。

### 3.1 UI自动化与DOM树拉取 (UITools)

这是整个方案的**技术关键**，解决了AI对应用UI的感知问题。实现方案是**在目标应用中集成一个轻量级的HTTP服务器，该服务器利用UITest框架的API来遍历UI控件并返回UI树结构**。

**Tool: `get_ui_tree`**

- **功能**：获取当前前台应用的UI树结构。
- **实现**：
    1.  通过`hdc`命令获取当前前台应用的`bundleName`。
    2.  通过`hdc`的端口转发功能，将设备上的HTTP服务端口映射到本地。
    3.  MCP工具向本地端口发送HTTP请求。
    4.  应用内的HTTP服务接收请求，调用`UITest`的`Driver.findComponent(ON.type('Page'))`获取根节点，并递归遍历所有子控件，获取其`id`、`type`、`text`、`bounds`等属性。
    5.  将遍历结果构建成JSON格式的UI树，通过HTTP响应返回。

```python
# mcp_server.py
from fastmcp import FastMCP
import requests

server = FastMCP('harmonyos-tools')

@server.tool()
def get_ui_tree(device_id: str = None) -> dict:
    """获取前台应用的UI树结构。"""
    # 1. 获取设备ID
    # 2. hdc port forward tcp:8080 tcp:8080
    try:
        response = requests.get('http://localhost:8080/get-ui-tree')
        response.raise_for_status()
        return {'success': True, 'tree': response.json()}
    except requests.RequestException as e:
        return {'success': False, 'error': str(e)}
```

**UI树JSON结构示例**：
```json
{
  "type": "Page",
  "id": "root",
  "bounds": {"x": 0, "y": 0, "width": 1080, "height": 2340},
  "children": [
    {
      "type": "Button",
      "id": "btn_login",
      "text": "登录",
      "bounds": {"x": 100, "y": 200, "width": 880, "height": 120},
      "attributes": {"clickable": true}
    }
  ]
}
```

**Tool: `find_element`, `click_element`, `input_text`**

- **功能**：基于UI树实现元素查找、点击和输入。
- **实现**：这些工具同样通过HTTP请求应用内的服务，应用端接收到请求后，使用`UITest`的`driver.findComponent(ON.id('element_id'))`找到控件并执行相应操作。

## 4. 实施与部署

### 4.1 MCP Server开发

1.  **环境准备**：安装Python，并使用`pip install fastmcp`安装框架。
2.  **工具实现**：创建`mcp_server.py`文件，按照第3节的定义实现所有`@server.tool()`。
3.  **命令行包装**：为`hdc`和`hvigorw`编写健壮的Python包装器，处理命令执行、超时和输出解析。
4.  **启动服务**：通过`python mcp_server.py`启动服务。

### 4.2 目标应用改造 (用于UI感知)

1.  **添加依赖**：在目标应用的`oh-package.json5`中添加`@kit.TestKit`和HTTP服务器库的依赖。
2.  **实现HTTP服务**：在应用启动时，初始化一个HTTP服务器，并提供`/get-ui-tree`等接口。
3.  **权限申请**：确保应用拥有必要的权限（如网络权限）。

### 4.3 AI IDE集成

以Cursor为例，在项目根目录下创建`.cursor/mcp.json`文件，配置本地MCP服务器的启动方式。

```json
// .cursor/mcp.json
{
  "mcpServers": {
    "harmonyos-tools": {
      "command": "python",
      "args": ["/path/to/your/mcp_server.py"],
      "env": {
        "HARMONYOS_SDK_PATH": "/path/to/harmonyos/sdk"
      }
    }
  }
}
```

配置完成后，重启Cursor，即可在聊天或代码编辑中通过`@harmonyos-tools`来调用已定义的工具。

## 5. 风险与挑战

- **HarmonyOS SDK版本迭代**：`hdc`和`hvigorw`的命令参数可能随SDK版本更新而变化，MCP工具需要保持同步更新和版本兼容性管理。
- **UITest框架的限制**：`UITest`可能无法获取某些自定义控件的全部属性，或在复杂UI结构下性能较低。需要设计备用方案（如图像识别）作为补充。
- **设备连接稳定性**：`hdc`的连接可能不稳定，特别是在使用Wi-Fi调试时。MCP工具需要加入重试和错误处理机制。
- **安全性**：在应用内开启HTTP服务存在安全风险，必须限制其仅在本地（`127.0.0.1`）监听，并考虑添加Token验证机制。

## 6. 结论

通过开发一个专为HarmonyOS设计的MCP工具，我们可以有效克服当前AI编程工具在HarmonyOS生态中的应用障碍。该方案以`hdc`、`hvigorw`和`UITest`为基础，通过MCP协议将底层的开发、构建和测试能力暴露给上层的AI Agent，从而构建起一套端到端的、自动化的AI辅助开发流程。方案的核心和难点在于实现稳定可靠的UI树获取机制，通过在应用内集成基于`UITest`的HTTP服务是当前最为可行的路径。
